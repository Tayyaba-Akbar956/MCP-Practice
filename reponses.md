<div class=\"md-content\" data-md-component=\"content\">\n              <article class=\"md-content__inner md-typeset\">\n                \n                  \n\n\n  \n  \n\n\n<h1 id=\"openai-agents-sdk\">OpenAI Agents SDK</h1>\n<p>The <a href=\"https://github.com/openai/openai-agents-python\">OpenAI Agents SDK</a> enables you to build agentic AI apps in a lightweight, easy-to-use package with very few abstractions. It's a production-ready upgrade of our previous experimentation for agents, <a href=\"https://github.com/openai/swarm/tree/main\">Swarm</a>. The Agents SDK has a very small set of primitives:</p>\n<ul>\n<li><strong>Agents</strong>, which are LLMs equipped with instructions and tools</li>\n<li><strong>Handoffs</strong>, which allow agents to delegate to other agents for specific tasks</li>\n<li><strong>Guardrails</strong>, which enable validation of agent inputs and outputs</li>\n<li><strong>Sessions</strong>, which automatically maintains conversation history across agent runs</li>\n</ul>\n<p>In combination with Python, these primitives are powerful enough to express complex relationships between tools and agents, and allow you to build real-world applications without a steep learning curve. In addition, the SDK comes with built-in <strong>tracing</strong> that lets you visualize and debug your agentic flows, as well as evaluate them and even fine-tune models for your application.</p>\n<h2 id=\"why-use-the-agents-sdk\">Why use the Agents SDK</h2>\n<p>The SDK has two driving design principles:</p>\n<ol>\n<li>Enough features to be worth using, but few enough primitives to make it quick to learn.</li>\n<li>Works great out of the box, but you can customize exactly what happens.</li>\n</ol>\n<p>Here are the main features of the SDK:</p>\n<ul>\n<li>Agent loop: Built-in agent loop that handles calling tools, sending results to the LLM, and looping until the LLM is done.</li>\n<li>Python-first: Use built-in language features to orchestrate and chain agents, rather than needing to learn new abstractions.</li>\n<li>Handoffs: A powerful feature to coordinate and delegate between multiple agents.</li>\n<li>Guardrails: Run input validations and checks in parallel to your agents, breaking early if the checks fail.</li>\n<li>Sessions: Automatic conversation history management across agent runs, eliminating manual state handling.</li>\n<li>Function tools: Turn any Python function into a tool, with automatic schema generation and Pydantic-powered validation.</li>\n<li>Tracing: Built-in tracing that lets you visualize, debug and monitor your workflows, as well as use the OpenAI suite of evaluation, fine-tuning and distillation tools.</li>\n</ul>\n<h2 id=\"installation\">Installation</h2>\n<div class=\"language-bash
